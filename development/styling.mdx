## Overview

Los Simpson uses **CSS Modules** for component-level styling, combined with global CSS for app-wide styles and design tokens. This approach provides:

- **Scoped styles** - No CSS class name conflicts
- **Type safety** - Import styles like JavaScript modules
- **Maintainability** - Co-locate styles with components
- **Performance** - Only load styles when components are used

## Global Styles

Global styles and CSS custom properties are defined in `src/styles/global.css`.

### CSS Custom Properties

```css title="src/styles/global.css"
:root {
    --bg-primary: radial-gradient(circle, rgba(61, 228, 244, 1) 0%, rgba(1, 92, 158, 1) 100%);
    --text-primary: #fff;
    --bg-button: #FED90F;
    --text-button: #000;
    --text-secondary: #000;
    --bg-button-hover: #e1c41d;
    --bg-article: #e8f8ff;
    --loading: #ffda05;
    --global-hover: #545454;

```

<Note>
  CSS custom properties allow for consistent theming across all components. These variables represent The Simpsons' iconic color palette.
</Note>

## Using CSS Modules in Components

CSS Modules are imported as JavaScript objects. Class names are accessed as properties.

### Import Pattern

```typescript title="src/pages/charathers.tsx"
import CardStyle from './../styles/charather.module.css'
```

### Usage in JSX

```tsx title="src/pages/charathers.tsx"
export default function Characters() {
    return (
        <main>
            <div className={CardStyle.container}>
                <div className={CardStyle.cards}>
                    <article className={CardStyle.card}>
                        <div className={CardStyle.image}>
                            <h2 className={CardStyle.name}>Character Name</h2>
                            <div className={CardStyle.imageContainer}>
                                {/* Image component */}
                            </div>
                        </div>
                        <div className={CardStyle.info}>
                            <p className={CardStyle.alive}>Alive</p>
                        </div>
                    </article>
                </div>
            </div>
        </main>
    )
}
```

### Conditional Classes

```tsx
<p className={character.status === 'Alive' ? CardStyle.alive : CardStyle.dead}>
    {character.status === 'Alive' ? 'Alive' : 'Deceased'}
</p>
```

### Hover Effects

All interactive elements have smooth hover transitions:

```css
.card {
    border: transparent 3px solid;
    transition: border 200ms ease-out;
}

.card:hover {
    border: var(--global-hover) 3px solid;
}
```

### Image Handling

```css
.imageContainer {
    width: 150px;
    height: 150px;
    overflow: hidden;
}

.card img {
    aspect-ratio: 1 / 1;
    object-fit: cover;
    width: 150px;
}
```

<Tip>
  Using `aspect-ratio` and `object-fit: cover` ensures images maintain consistent dimensions while preserving their aspect ratio.
</Tip>

## Best Practices

## File Organization

```plaintext
src/
├── styles/
│   ├── global.css               # Global styles and variables
│   ├── charather.module.css     # Character components
│   ├── episode.module.css       # Episode components
│   ├── nav.module.css           # Navigation
│   ├── pageCounter.module.css   # Pagination
│   ├── loading.module.css       # Loading states
│   ├── explorer.module.css      # Explorer page
│   ├── app.module.css           # App layout
│   └── 404.module.css           # Error page
```

<Note>
  Each page or major component has its own dedicated CSS Module file, making styles easy to locate and maintain.
</Note>

## Loading Global Styles

Global styles are loaded in the main HTML file:

```html title="index.html"
<head>
  <link rel="stylesheet" href="./src/styles/global.css"/>
</head>
```
