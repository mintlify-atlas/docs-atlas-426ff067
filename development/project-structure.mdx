## Directory Structure

The Los Simpson project follows a clean and organized structure typical of modern React + TypeScript applications:

```plaintext
los-simpson/
├── public/              # Static assets
│   └── vite.svg        # Vite logo
├── src/                # Source code
│   ├── assets/         # Images, fonts, and other media
│   │   ├── fonts/      # Custom font files
│   │   └── images/     # Image assets
│   ├── components/     # React components
│   │   ├── data/       # Data-related utilities
│   │   ├── icons/      # Icon components
│   │   └── ui/         # UI components
│   ├── pages/          # Page components (routes)
│   ├── styles/         # CSS Modules
│   ├── main.tsx        # Application entry point
│   └── types.d.ts      # TypeScript type definitions
├── index.html          # HTML entry point
├── package.json        # Dependencies and scripts
├── tsconfig.json       # TypeScript configuration
├── tsconfig.app.json   # App-specific TypeScript config
├── tsconfig.node.json  # Node-specific TypeScript config
├── vite.config.ts      # Vite configuration
├── eslint.config.js    # ESLint configuration
└── vercel.json         # Vercel deployment config
```

## Source Directory Structure

### `/src/assets/`
Contains all static assets used in the application:

- **`fonts/`** - Custom web fonts (Funnel Sans, Concert One)
- **`images/`** - Image files (family picture, logo)
- **`index.ts`** - Central export file for assets

```plaintext
assets/
├── fonts/
│   ├── ConcertOne-Regular.woff2
│   ├── FunnelSans-VariableFont_wght.woff2
│   └── index.css
├── images/
│   ├── Simpsons_FamilyPicture.png
│   └── logo.png
├── react.svg
└── index.ts
```

### `/src/components/`
Reusable React components organized by purpose:

```plaintext
components/
├── data/
│   └── explorer.ts          # Data exploration utilities
├── icons/
│   └── Loading.tsx          # Loading icon component
├── ui/
│   ├── ImageWithLoading.tsx # Image component with loading state
│   ├── Loading.tsx          # Loading spinner component
│   ├── Page.Characters.tsx  # Character-specific page component
│   └── Page.Counter.tsx     # Pagination counter component
├── nav.tsx                  # Navigation component
└── index.ts                 # Component exports
```

<Note>
  Components are grouped by functionality: `data/` for data utilities, `icons/` for icon components, and `ui/` for user interface components.
</Note>

### `/src/pages/`
Page-level components representing different routes:

```plaintext
pages/
├── 404.tsx          # Not found page
├── charathers.tsx   # Characters browsing page
├── episodes.tsx     # Episodes browsing page
├── explorer.tsx     # Explorer page
└── index.tsx        # Home page
```

### `/src/styles/`
CSS Modules for component-specific styling:

```plaintext
styles/
├── 404.module.css           # 404 page styles
├── app.module.css           # Main app styles
├── charather.module.css     # Character card styles
├── episode.module.css       # Episode card styles
├── explorer.module.css      # Explorer page styles
├── global.css               # Global styles and CSS variables
├── loading.module.css       # Loading component styles
├── nav.module.css           # Navigation styles
└── pageCounter.module.css   # Pagination styles
```

<Tip>
  All component-specific styles use CSS Modules (`.module.css`) for scoped styling, while `global.css` contains global styles and CSS custom properties.
</Tip>

## Key Files

### `src/main.tsx`
The application entry point that renders the React app.

### `src/types.d.ts`
Central location for all TypeScript type definitions used throughout the application. See the [TypeScript documentation](/development/typescript) for details.

## Build Output

When you run `npm run build`, the following happens:

<Steps>
  <Step title="TypeScript Compilation">
    TypeScript compiles and type-checks all `.ts` and `.tsx` files
  </Step>
  <Step title="Vite Build">
    Vite bundles the application for production
  </Step>
  <Step title="Output to dist/">
    Optimized static files are generated in the `dist/` directory
  </Step>
</Steps>

<Warning>
  The `dist/` directory is git-ignored and should never be committed to version control.
</Warning>
